# Phase 6: Personalized Tool Recommendations & Social Sentiment

**Status:** Planning
**Created:** 2026-04-12
**Jira Epic:** KAN-6
**Scope:** Deep personalization layer for tool recommendations — market sentiment, social proof, honest analysis

---

## Overview

Transform the plan page from a basic tool matcher into a personalized recommendation engine. Users get light enrichment inline (sentiment badge + quick verdict) and can click through to a full deep-dive report with AI-synthesized insights from Reddit, X, Quora, and G2.

**Goal:** Make RightAIChoice the place users come back to before buying any AI tool — not just a discovery engine, but a decision engine.

---

## Architecture

### Two-Layer Design

**Layer 1 — Plan Page Enrichment (light)**
- Sentiment badge on each tool card (positive / mixed / negative)
- One-line AI verdict ("Best for solo creators who need quick video editing")
- "See full report" link to deep-dive page

**Layer 2 — Deep-Dive Report Page (`/tools/[slug]/report`)**
- Full 10-section personalized report
- Nested under existing tool detail page, shares data fetching
- Hybrid caching: scrape on first request, cache in Supabase for subsequent users

### Data Flow

```
User clicks "See full report"
  -> Check Supabase for cached sentiment data
  -> If cached & fresh (< 7 days): serve immediately
  -> If stale or missing:
      -> Trigger scraping jobs (Reddit, X, Quora, G2) in parallel
      -> Pass raw scraped content to Claude for synthesis
      -> Store synthesized report in Supabase
      -> Serve to user
  -> Show loading skeleton while scraping (first-time: ~10-15s)
```

## Report Page — 10 Sections

### 1. AI Verdict
One paragraph, honest, no-fluff assessment. Who this tool is for, what it does best, where it falls short. Generated by Claude from tool data + scraped social content. Displayed prominently at the top with a confidence indicator.

### 2. Five Pros / Five Cons
AI-synthesized from aggregated social posts across all sources. Each point is a concise, specific statement (not generic). Styled as two columns: green checkmarks (pros) vs red crosses (cons). Based on real user complaints and praise, not marketing copy.

### 3. Market Sentiment Score
Overall score: positive / mixed / negative. Per-source breakdown (Reddit, X, Quora, G2) with percentage bars. Sample size indicator ("Based on 47 mentions across 4 platforms").

### 4. What People Are Saying
3-5 themed summaries distilled from social data. Examples: "Users love the API but hate the pricing." Each theme tagged with which platform(s) it came from.

### 5. Best For / Not For
Specific user segments and use cases. Equally specific "Not For" section. Derived from social data + tool capabilities.

### 6. Pricing Breakdown
Detailed tier comparison table. Hidden costs flagged. Value-for-money verdict. Comparison to category average pricing.

### 7. Alternatives Comparison
Top 3-5 alternatives with quick compare table. Columns: pricing, key differentiator, sentiment score, best for. Links to each alternative's tool page and report.

### 8. Community Buzz
Recent mention volume (trending up/down/stable). Key topics being discussed. Sentiment trend over time.

### 9. Learning Curve
Time to get started. Skill requirements. Key hurdles users mention. Comparison to alternatives in terms of ease of use.

### 10. Integration Fit
How it plays with other tools in the user's planned stack. Known integrations + community-reported experiences. Context-aware when user came from a plan.

## Database Schema

### New table: `tool_sentiment_cache`

```sql
CREATE TABLE tool_sentiment_cache (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  tool_id UUID NOT NULL REFERENCES tools(id) ON DELETE CASCADE,

  -- Synthesized report data
  ai_verdict TEXT,
  pros JSONB,
  cons JSONB,
  sentiment_score TEXT,       -- "positive" | "mixed" | "negative"
  sentiment_breakdown JSONB,  -- {"reddit": 0.72, "twitter": 0.65, "quora": 0.80, "g2": 0.55}
  themes JSONB,
  best_for JSONB,
  not_for JSONB,
  pricing_analysis JSONB,
  community_buzz JSONB,
  learning_curve JSONB,
  integration_insights JSONB,

  -- Raw scraped data (for re-synthesis)
  raw_reddit JSONB,
  raw_twitter JSONB,
  raw_quora JSONB,
  raw_g2 JSONB,

  -- Metadata
  mention_count INT DEFAULT 0,
  sources_scraped JSONB,
  scraped_at TIMESTAMPTZ DEFAULT NOW(),
  synthesized_at TIMESTAMPTZ DEFAULT NOW(),
  expires_at TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '7 days'),

  UNIQUE(tool_id)
);

CREATE INDEX idx_sentiment_tool ON tool_sentiment_cache(tool_id);
CREATE INDEX idx_sentiment_expires ON tool_sentiment_cache(expires_at);
```

## API Routes

| Route | Method | Purpose |
|-------|--------|---------|
| `/api/tools/[slug]/report` | GET | Check cache, serve report or return `{ status: "generating" }` |
| `/api/tools/[slug]/report/generate` | POST | Scrape + synthesize + store (internal) |
| `/api/tools/[slug]/report/status` | GET | Polling endpoint: `generating` / `ready` / `failed` |

---

## Scraping Strategy

| Source | Method | Volume | Notes |
|--------|--------|--------|-------|
| Reddit | Reddit JSON API | 20-30 posts/comments | Target: r/SaaS, r/startups, r/Entrepreneur, r/artificial |
| X / Twitter | Apify Twitter scraper | 20 mentions (30 days) | Top by engagement |
| Quora | Apify web scraper | 10-15 answers | Search: review OR experience OR alternative |
| G2 | Apify web scraper | 20 reviews | Sorted by helpfulness |

**Caching:** Max 1 scrape per tool per 7 days. Fallback to available sources if any fail.

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Scraping sources block us | Rotate user agents, Apify proxies, fallback to available sources |
| Claude output inconsistency | Strict JSON schema, validation, retry on parse failure |
| Slow first-load experience | Loading skeleton, progress indicators, pre-scrape popular tools |
| Stale data feels misleading | "Last updated X days ago" + manual refresh button |
| API costs (Claude + Apify) | Aggressive caching (7-day TTL), rate limiting, pre-scrape only top 100 |
